面试-MyBatis常见面试题

发表于 2020-10-30 | 分类于 面试

1. #{}和${}区别

#{}和${}都是MyBatis中实现动态SQL的机制。

  • ${}是Properities文件中的占位符，用于标签属性值和SQL内部，属于静态文本替换。
  • #{}是参数占位符，#{}会解析为JDBC预编译PreparedStatement语句的参数标记符，一个 #{ } 被解析为一个参数占位符 ?。
  • ${} 在预编译之前已经被变量替换了，存在 sql 注入问题，而#{}会被预编译为？，在执行sql前按序设置参数值。

2. SQL预编译

sql 预编译指的是数据库驱动在发送 sql 语句和参数给 DBMS 之前对 sql 语句进行编译，这样 DBMS 执行 sql 时，就不需要重新编译。

JDBC 中使用对象 PreparedStatement 来抽象预编译语句

  • 预编译可以优化SQL执行，数据库不需要再次编译SQL语句
  • 预编译语句对象可以重复利用，SQL语句预编译后的PreparedStatement对象可以被缓存，不需要重复编译。

3. MyBatis中的一二级缓存

MyBatis的一二级缓存指的是执行查询的时候先从二级缓存中进行查询，若没有进入一级缓存中查询，若还是没有才执行JDBC查询。

  • 一级缓存(本地缓存)

    SqlSession级别的缓存。一级缓存是一直开启的，同一次会话期间从MySQL中查询到的数据会缓存在SqlSession的一个Map中，如果以后需要获取相同的数据，直接可以从缓存中取，不需要再查询数据库。

    一级缓存失效情况：

    • sqlSession不同
    • sqlSession相同，但是查询条件不同(当前一级缓存中没有这个数据)
    • sqlSession相同，两次查询之间执行了增删改操作（这次增删改可能对当前数据有影响）
    • sqlSession相同，手动清除了一级缓存（缓存清空）

  • 二级缓存(全局缓存)

    基于namespace的缓存，一个namespace对应一个二级缓存。MyBatis默认没有开启二级缓存。

    一个会话中查询的数据会被缓存在一级缓存中，如果会话关闭，一级缓存中的数据会被保存到二级缓存中，

    不同namespace查出的数据会保存在自己的缓存中。

默认的MyBatis Cache都是基于本地的，分布式环境下会出现缓存数据不一致，建议使用第三方集中式缓存或者使用Redis等进行缓存。

4. MyBatis如何防止SQL注入

SQL注入是一种代码注入技术，通过把SQL命令插入到Web表单提交或页面请求url的查询字符串，最终达到欺骗服务器执行恶意的SQL命令。

MyBatis通过预编译加上#{}参数占位符来防止SQL注入。#{}在预编译中会被替换为？，执行SQL前会按序进行参数替换。

1. Object类方法

Java中Object类是所有类的顶层父类，所以Java中的任何类都会继承Object类中的13个方法：

  • Object()

    默认构造方法

  • clone()

    默认创建一份对象的拷贝，Object类的clone方法是一个protected的native方法，所以需要继承Cloneable接口。

  • equals()

    比较两个对象是否相等。

    默认情况下，对于基本数据类型，比较的是值是否相等；对于引用数据类型，判断的是内存地址是否相同。

    可以按照特定的逻辑重写此方法来判断两个对象是否相等。

  • finalize()

    是一个protected方法，在实例被垃圾回收前被调用。默认是一个空方法。

  • getClass()

    是一个本地的final方法，不允许子类重写，返回当前运行时对象的CLass对象。

  • hashCode()

    返回对象的哈希码，一般在哈希表中用到。必须同时重写一个类的hashCode()和equals()方法。

    • 如果两个对象相等，则哈希码一定相等；
    • 如果两个对象哈希码相等，这两个对象不一定相等。

    所以，重写equals必须重写hashCode.

发表于 2020-10-27 | 分类于 面试

1. 进程与线程的区别

进程是程序的一次执行过程，是操作系统中资源分配的最小单位，进程间相互独立，进程的进程控制块pcb描述了进程的基本信息和运行状态；

线程是独立调度的基本单位，一个进程中可以有多个线程，它们共享进程的资源。

  • 拥有资源

    进程是资源分配的基本单位，同一个进程下的多个线程共享进程的资源

  • 调度

    线程是独立调度的基本单位，同一进程中，线程切换不会引起进程切换

  • 系统开销

    创建或撤销进程时，系统都要分配或回收资源，开销远大于线程；

    执行进程切换时涉及虚拟地址的切换，虚拟地址的切换需要页表的切换，将虚拟地址转化为物理地址需要查找页表，页表查找很慢，通常是使用Cache来缓存页表映射，页表的切换会导致Cache失效需要重新建立映射，开销较大，而线程切换只需要设置少量寄存器，上下文切换成本进程远大于线程。

    所以，线程被称为轻量级进程。

  • 通信

    线程间通信可以通过直接读写进程的共享数据，进程间通信需要借助IPC。

面试总结-MySQL

发表于 2020-10-24 | 分类于 面试

1. 数据库引擎MyISAM和InnoDB区别

MyISAM在5.5以前是MySQL的默认数据库引擎，支持全文索引、压缩和空间函数等，但MyISAM不支持事务和行级锁，并且崩溃后无法安全恢复。5.5之后，MySQL默认使用InnoDB作为默认数据库引擎。

对比：

  • 是否支持行级锁：MyISAM只支持表级锁，而InnoDB支持表级锁和行级锁，默认为行级锁；
  • 是否支持事务和崩溃后安全恢复：MyISAM追求查询效率，不支持事务；而InnoDB提供事务支持，具有事务回滚和崩溃后修复能力。
  • 是否支持外键：MyISAM不支持，InnoDB支持
  • 是否支持MVCC：只有InnoDB支持
  • MyISAM支持全文索引，InnoDB不支持
  • MyISAM相对简单，适合读密集场景；而InnoDB适合大型读写高并发场景。

1. 跨域

跨域问题是由浏览器同源策略引起的。同源策略是浏览器的一种安全机制，主要是为了防止XSS和CSRF攻击。

同源策略指的是协议、域名、端口不同时，对Cookie、DOM节点和AJAX请求会被浏览器拦截。

跨域请求能够发送出去，服务端也能收到请求并返回结果，只是结果被浏览器拦截了。

可以通过cors解决跨域问题，服务端设置Access-Control-Allow-Origin字段就可以开启cors。该属性表示那些域名可以访问资源，设置成通配符表示所有网站都可以访问资源。

cors将请求分为简单请求和复杂请求。

不安全的HashMap

总所周知，HashMap是非线程安全的，体现在rehash时的死循环，使用迭代器时的fast-fail和多线程环境下put操作出现丢失数据等。

虽然JDK也提供了一个安全版本的HashMap：Collections.SynchronizedMap，但因为其使用的是全局锁，并发性能较差。

因此，juc包下提供了线程安全且锁粒度较小的ConcurrentHashMap。

Java多线程-锁

发表于 2020-09-10 | 分类于 Java多线程

Java中的锁

本篇文章主要介绍juc包下与锁相关的API和组件，包括Lock接口、AQS抽象队列同步器、重入锁、读写锁和Condition接口。

Lock接口

锁是用来控制多个线程访问资源的方式，除了synchronized之外，juc包下还提供了Lock接口来实现锁功能。其提供了与synchronized类似的同步功能，但需要显示的获取、释放锁，并且提供了更加强大的功能，如可中断的获取锁和超时获取锁等。

Lock接口提供了synchronized关键字不具备的特性：

特性 描述
尝试非阻塞的获取锁 当前线程尝试获取锁，如果锁这一时刻没有被其他线程获取到，则成功获取并持有锁
可中断的获取锁 获取到锁的线程能响应中断，拥有锁的线程被中断时，会抛出中断异常并释放锁
超时获取锁 在指定时间内获取锁，如果超时仍未获取锁，则返回

Lock接口的默认实现类为ReentrantLock，部分API如下：

void lock() 获取锁，调用该方法的线程尝试获取锁，获取成功后返回
void lockInterruptibly() throws InterruptedException 可中断的获取锁
boolean tryLock() 尝试非阻塞的获取锁，调用此方法后立即返回
boolean tryLock(long var1, TimeUnit var3) throws InterruptedException 超时获取锁，有三种可能会返回：超时、成功获取锁、被中断
void unlock() 释放锁
Condition newCondition() 获取等待通知组件，该组件和当前的锁绑定，只有获取了锁才能调用此方法，并且调用后释放锁
